6. Date Edtf

Label Date Edtf
URI [baseURI-toBeDetermined]/dateEdtf
Dublin Core Equivalent http://purl.org/dc/elements/1.1/created
CONTENTdm searchable? no
CONTENTdm hidden? yes
Required by CONTENTdm? no
CONTENTdm controlled Vocabulary? no
Required by UW Libraries? yes
Definition

Prominent date for the resource, such as the date of creation of the original resource.
Input instructions

Reflects the date in the "Date of Broadcast" field.

Express all dates, including date ranges, using Library of Congress' Extended Date Time Format.

Most recent version of EDTF available at https://www.loc.gov/standards/datetime/.

Six formats are recommended for values in Date of Broadcast, each corresponding to a specific EDTF format:

If the value of Date of Broadcast is YYYY, enter YYYY.

If the value of Date of Broadcast is YYYY?, enter YYYY?

If the value of Date of Broadcast is [month] DD, YYYY, enter YYYY-MM-DD.

If the value of Date of Broadcast is [month], YYYY, enter YYYY-MM.

If the value of Date of Broadcast is between YYYY and YYYY, enter YYYY/YYYY

If the value of Date of Broadcast is between YYYY and YYYY?, enter YYYY?/YYYY?

If the value of Date of Broadcast is not any of the above, consult the EDTF spedification to determine the appropriate notation.

Date-Edtf value should agree with dates values entered in Date of Broadcast and Dates fields.

Input examples

If the Date of Broadcast field = "1940" then Date EDTF = "1940"

If the Date of Broadcast field = "1910?" then Date EDTF = "1910?"

If Date of Broadcast field = "May 9, 2001" then Date EDTF = "2001-05-09"

If Date of Broadcast field = "January 2008" then Date EDTF = "2008-01"

If Date of Broadcast field = "between 1930 and 1939" then Date EDTF = "1930/1939"

If Date of Broadcast field = "between 1855 and 1999?" then Date EDTF = "1855?/1999?"
Additional notes

For the xsd, datatype is xs:string; the datatype is more accurately http://id.loc.gov/datatypes/edtf, which is not a valid QName and is inavlid according to schema-4-properties, but this does not seem to be machine-actionable; more exploration needed.

8. Dates

Label Dates
URI [baseURI-toBeDetermined]/datesAll
Dublin Core Equivalent [no Dublin Core equivalent]
CONTENTdm searchable? yes
CONTENTdm hidden? yes
Required by CONTENTdm? no
CONTENTdm controlled Vocabulary? no
Required by UW Libraries? yes
Definition

Prominent years for the resource, such as the date(s) of creation of the original resource; the expected value is a list of each and every pertinent year.

Input instructions

Reflects the date in the "Date of Broadcast" field.

Enter the year or years only.

When the value of Date of Broadcast is YYYY, enter YYYY.

When the value of Date of Broadcast is YYYY?, enter an eleven year date range, five years on either side of the uncertain date.

When the value of Date of Broadcast is [month] DD, YYYY, enter the year only in the following format: YYYY.

When the value of Date of Broadcast is [month], YYYY, enter the year only in the following format: YYYY.

When the value of Date of Broadcast is between YYYY and YYYY, enter all years in the date range in the following format: YYYY YYYY YYYY etc.

When the value of Date of Broadcast is between YYYY and YYYY?, enter all years in the date range plus five years before the earliest date and five years after the latest date, in the following format: YYYY YYYY YYYY etc.

This field is indexed and enables date searching in CONTENTdm.

The Dates value should agree with date values entered in Date-Edtf and Date of Broadcast fields.

Input examples

1939?
